# 抄表录入弹窗 UI 重设计

## 背景

水电抄录页面（`Hans/src/pages/utility/meter/index.vue`）的"抄表录入"弹窗当前存在**高度过高、超出屏幕底部**的问题。

现行弹窗宽 `600px`，内容采用纯纵向堆叠，包含 4 条 `<t-divider>` 分割标题（上次读数 / 本次读数 / 用量计算 / 固定费用），每块各占独立区块。即便字段间已使用两列 `t-row`/`t-col` 布局，整体仍堆叠出约 13 行可视高度，且弹窗未设置 `max-height`，内容增多时直接顶出视口，底部按钮不可见。

## 需求

重新设计弹窗布局，**首要目标是把弹窗压缩到一屏内可见**（1366×768 笔记本即可容纳），其次保持信息分层清晰、交互流程顺畅。

### 约束

- 仅改动"抄表录入"弹窗的 UI 布局，不改动提交逻辑、校验规则、计算逻辑
- 沿用现有技术栈：Vue 3 + TDesign Vue Next + TypeScript
- 字段集合不变：房间、日期、上次水/电读数、本次水/电读数、用水/电量、水/电单价、水/电小计、固定费用、合计、备注
- 现有交互保留：实时用量计算、超阈值红字警告（>50 吨 / >200 度）、单价取自房间配置
- 仅适配桌面端（≥1280px），不做手机端适配

## 设计

### 方案选型

在"表格对照式 / 左右分栏式 / 纵向紧凑式"三者中选择 **表格对照式**。

结构为一个 3 列对照表：左列为指标名（上次/本次/用量/单价/小计），中列为水，右列为电。水/电两条线天然并排对照，通过表格行分隔取代 4 条 `t-divider`，垂直空间最省（约 9 行）。

选择理由：水/电数据结构对称，对照表让两条线的 6 个指标逐行对齐，既压缩高度又强化可读性；左右分栏式右栏仍偏高，纵向紧凑式在 600px 下压一屏较勉强。

### 弹窗骨架

```
┌─ 抄表录入 ───────────────────────────────────────┐
│ 房间 [▼ 选择房间]            日期 [📅 2026-07-02] │   ← 并排一行（原各占整行）
├──────┬─────────────────┬───────────────────────┤
│      │   💧 水         │   ⚡ 电               │
│ 上次 │ 1234.00         │ 5678.00               │   只读灰字
│ 本次 │ [______]        │ [________]            │   唯一输入项
│ 用量 │  50 吨          │  200 度 ⚠            │   >50吨/>200度 红字
│ 单价 │ ¥5.00/吨        │ ¥1.00/度              │   未设置显示"未设置"
│ 小计 │ ¥250.00         │ ¥200.00               │
├──────┴─────────────────┬───────────────────────┤
│ 固定费用                │  合计费用              │
│ 电梯 ¥50 · 物业 ¥30     │  ¥540.00             │   ← 并排一行
│ 网络 ¥40 · 其他 ¥20     │  （大号字+强调色）     │
├────────────────────────┴───────────────────────┤
│ 备注 [______________________________________]   │   textarea autosize 1-3 行
└─────────────────────────────────────────────────┘
```

### 前端变更

#### 弹窗容器（`Hans/src/pages/utility/meter/index.vue`，约 93-243 行）

- 宽度由 `600px` 改为 `820px`
- 新增 `max-height: 85vh`，仅 `<t-form>` 内容区可滚动（header / footer 固定），通过给 `t-dialog` 的 body 设置自定义 class + scoped less `:deep()` 实现，避免全局污染：
  ```less
  :deep(.meter-dialog__body) {
    max-height: calc(85vh - 140px);
    overflow-y: auto;
  }
  ```
- 删除全部 4 条 `<t-divider>` 分割标题

#### 顶部：房间 + 日期并排

- 用单个 `<t-row :gutter="24">` 包裹两个 `<t-col :span="12">`，分别放"选择房间"和"抄表日期"
- 合并为一行，省 1 行

#### 表格主体：水/电双列对照

用一个 3 列结构（指标名 / 水 / 电）承载 6 行，取代原"上次读数 / 本次读数 / 用量计算"三块：

| 行 | 列-水 | 列-电 |
|----|-------|-------|
| 上次（只读灰字） | `lastReadings.waterReading` | `lastReadings.electricReading` |
| 本次（`t-input-number` 输入，`name` 校验保留） | `formData.waterReading` | `formData.electricReading` |
| 用量（超阈值红字警告保留） | `calculatedUsage.water` 吨 | `calculatedUsage.electric` 度 |
| 单价（取自 `selectedRoom`，未设置显示"未设置"） | `selectedRoom.waterPrice` /吨 | `selectedRoom.electricPrice` /度 |
| 小计 | `calculatedFees.water` | `calculatedFees.electric` |

表格实现采用原生语义化 `<table>`（行高紧凑、列对齐天然，且不依赖 TDesign 栅格的 gutter/响应式）。表头行显示"💧 水 / ⚡ 电"，首列右对齐标签、后两列左对齐数值。输入行（本次读数）内嵌 `<t-input-number>`，校验 `name` 仍挂在该行对应单元上。

#### 底部：固定费用 + 合计（并排）+ 备注

- **固定费用**（`v-if="hasFixedFees"`）：改为横排流式标签，仅显示已配置项，用 `·` 分隔，取代原四宫格 `t-col :span="6"`：
  `电梯 ¥50 · 物业 ¥30 · 网络 ¥40 · 其他 ¥20`
- **合计费用**：与固定费用左右并排（各 `span=12`），放右侧；沿用 `.total-fee-large` 样式（18px / 600 字重），加强调底色块作为视觉锚点
- **备注**：沿用 `t-textarea`，autosize 由 `{minRows:2, maxRows:4}` 收紧为 `{minRows:1, maxRows:3}`

#### 表单逻辑（不变）

以下均保持现状，仅在模板中重新引用到对照表行：
- `formRules`（roomId / meterDate / waterReading / electricReading 必填）
- `calculatedUsage`（本次 − 上次，小于上次为 0）
- `calculatedFees`（用量 × 单价 + 固定费用求和）
- `handleRoomChange`（拉取上次读数）
- `handleSubmit`（校验、防读数倒退、提交）

#### 样式（`<style lang="less" scoped>`，约 622-689 行）

保留：`.prev-reading`、`.usage-warning`、`.fee-text`、`.total-fee-large`。
新增：
- 对照表相关 class（如 `.meter-table`、行间距、表头样式）
- 弹窗 body 滚动约束（`:deep(.meter-dialog__body)`）
- 合计费用强调底色块（如 `.total-fee-block`）

### 兜底机制

- `max-height: 85vh` + 内容区滚动，确保任意房间配置下弹窗不顶出视口，header/footer 始终可见
- 宽度策略：≥1440px 舒展，1280-1440px 两侧留白收窄至 ~230px 仍舒适，<1280px 触发滚动兜底
- 用量超阈值红字警告不受布局变化影响，继续生效

## 高度估算

| 区块 | 行数 |
|------|------|
| 顶部（房间+日期并排） | 1 |
| 表格主体（6 行指标） | 6 |
| 底部（固定费用+合计并排） | 1 |
| 底部（备注 1-3 行） | 1-3 |
| **合计** | **约 9-11 行** |

较现行约 13 行减少约 40%，1366×768 笔记本可一屏容纳。
