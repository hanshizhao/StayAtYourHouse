# 房源管理模块设计

## 模块概述

管理小区和房间信息，记录租入成本和出租定价，支持退租结算计算。

## 小区信息

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| name | string | 小区名称 |
| address | string | 详细地址 |
| property_phone | string | 物业电话 |
| remark | string | 备注 |

## 房间信息

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| community_id | int | 所属小区ID |
| building | string | 楼栋 |
| room_number | string | 房号 |
| area | decimal | 面积（平方米，可选） |
| room_type | string | 房间类型（单间、一室一厅等） |
| cost_price | decimal | 租入成本（元/月） |
| rent_price | decimal | 出租定价（元/月） |
| deposit | decimal | 押金金额 |
| water_price | decimal | 水费单价（元/吨），可单独设置 |
| electric_price | decimal | 电费单价（元/度），可单独设置 |
| status | enum | 状态：vacant(空置)/rented(已出租)/renovating(装修中) |
| contract_image | string | 合同图片路径（当前租客的合同） |
| remark | string | 备注 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

## 状态流转

```
空置 ──(入住)──> 已出租 ──(退租)──> 空置
  │                  │
  └──(装修)──> 装修中 ──(完工)──> 空置
```

## 退租结算

**触发时机**：房客退租时

**结算公式**：
```
结算金额 = 剩余房租 + 押金 - 水电费
剩余房租 = 租金 / 30 × 剩余天数
剩余天数 = 从退租日到下次应收租日的天数
```

**结算金额含义**：
- 结算金额 > 0：退还给房客
- 结算金额 < 0：房客需补交

## 利润计算

- 单间月利润 = 出租定价 - 租入成本
- 可按小区/整体汇总利润

## 业务规则

1. 房间新增时默认状态为"空置"
2. 入住操作（租客模块）自动将状态改为"已出租"，同时可上传合同图片
3. 退租操作（租客模块）自动计算结算金额，确认后状态改为"空置"，合同图片清空
4. 装修状态手动切换
5. 合同图片仅保留当前租客的，新入住时覆盖
