# 二房东房源管理系统 - 项目梗概

## 项目背景

**项目名称**：二房东房源管理系统

**业务背景**：
房东作为二房东，管理约160间分散在多个小区的出租房源。目前使用纸质合同和Excel表格记录信息，存在以下痛点：

1. **收租混乱** - 按签约日收租，每天可能有人该交租，宽限、催收容易遗漏
2. **成本不清** - 房源租入成本和出租定价凭感觉，利润算不清楚
3. **信息分散** - 租客信息、合同、联系方式分散在多处，查找困难
4. **水电繁琐** - 需要人工抄表计算水电费，容易出错

**目标**：搭建一套信息化管理系统，实现房源、租客、收租、水电的一体化管理。

## 项目目标与成功标准

**核心目标**：
1. **收租不漏** - 系统自动提醒今天该收谁、谁宽限到期、谁逾期了
2. **成本清晰** - 每间房的租入成本、出租价格、利润一目了然
3. **信息集中** - 租客资料、合同、联系方式一处管理，随时可查
4. **水电自动** - 抄表后自动计算用量和费用，与房租合并收取

**成功标准**：
- 房东每天打开系统，能清楚看到"今天该做什么"
- 收租不再靠记忆和翻聊天记录
- 每间房的盈亏情况随时可查
- 5分钟内能查到任意租客的完整信息

**非目标**（本期不做）：
- 多用户权限系统
- 租客自助端
- 在线支付集成

## 技术架构

**前端**（Hans 项目）：
- Vue 3 + TypeScript + Vite
- UI 组件库：TDesign Vue Next
- 状态管理：Pinia
- 已有管理后台脚手架，直接在此基础上开发

**后端**（Gentle 项目）：
- .NET 10 + Furion 框架
- 数据库：MySQL
- ORM：Entity Framework Core
- 认证：JWT（单用户，简单登录即可）

**部署方式**：
- 前后端均容器化，运行在 Docker 中
- MySQL 也运行在 Docker 容器
- 使用 Docker Compose 编排服务
- 便于迁移和扩展

## 模块概览

| 模块 | 核心功能 | 优先级 |
|------|----------|--------|
| **房源管理** | 小区信息、房间信息、租入成本、出租定价 | P0 |
| **租客管理** | 租客资料、合同信息、入住/退租操作 | P0 |
| **收租管理** | 收租日提醒、宽限记录、催收跟踪、收款登记 | P0 |
| **水电抄表** | 抄表记录、用量计算、费用合并 | P1 |
| **统计报表** | 收入统计、利润分析、空置率、账单汇总 | P1 |

## 开发顺序

1. 先完成房源管理 + 租客管理（基础数据）
2. 再完成收租管理（核心业务）
3. 最后完成水电抄表 + 统计报表（增强功能）
