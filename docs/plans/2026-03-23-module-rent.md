# 收租管理模块设计

## 模块概述

管理收租账单、催收记录、宽限处理，提供首页待办提醒，确保收租不遗漏。

## 系统设置

**房租催收提醒配置**（按付款周期）：

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| monthly_remind_days | int | 月付提前催收天数 | 3 |
| quarterly_remind_days | int | 季付提前催收天数 | 7 |
| half_yearly_remind_days | int | 半年付提前催收天数 | 10 |
| yearly_remind_days | int | 年付提前催收天数 | 15 |

## 账单生成规则

**触发时机**：
- 入住时：自动生成第一期账单
- 收租成功时：自动生成下一期账单

**示例**：

| 场景 | 操作 |
|------|------|
| 6月5日入住，季付 | 自动生成第一期账单（9月5日应收） |
| 9月5日收租成功 | 自动生成第二期账单（12月5日应收） |
| 中途退租 | 无多余账单，直接结束 |

## 数据结构

### 账单信息

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| rental_record_id | int | 租住记录ID |
| period_start | date | 账单周期开始 |
| period_end | date | 账单周期结束 |
| due_date | date | 应收日期 |
| rent_amount | decimal | 房租 |
| water_fee | decimal | 水费 |
| electric_fee | decimal | 电费 |
| total_amount | decimal | 应收总额 |
| status | enum | 状态：pending(待收)/grace(宽限中)/paid(已收)/overdue(逾期) |
| paid_amount | decimal | 实收金额 |
| paid_date | date | 收款日期 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 催收记录

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| bill_id | int | 账单ID |
| collect_date | date | 催收日期 |
| result | enum | 结果：success(成功)/grace(宽限)/refuse(拒付) |
| grace_until | date | 宽限至日期（选择"宽限"时填写） |
| remark | string | 备注 |
| created_at | datetime | 创建时间 |

## 账单状态规则

| 状态 | 条件 |
|------|------|
| 待收 | 账单生成后默认状态 |
| 宽限中 | 有宽限记录且宽限期未到 |
| 已收 | 收款成功 |
| 逾期 | 超过应收日期7天且未收款 |

## 业务规则

1. **账单自动生成**：
   - 入住时根据付款周期生成第一期账单
   - 收租成功后自动生成下一期账单
   - 退租时无需清理（因为只生成一期）

2. **催收流程**：
   - 点击"催收"按钮打开催收弹窗
   - 选择催收结果并填写相关信息
   - "收款成功"时账单状态改为"已收"，同时生成下一期账单
   - "同意宽限"时账单状态改为"宽限中"，记录宽限日期
   - "拒付"时仅记录，账单状态不变

3. **状态变更**：
   - 宽限期到期后，账单状态自动恢复为"待收"
   - 超过应收日期7天，账单状态自动改为"逾期"
   - 收款成功后，账单状态改为"已收"
